## Reflection

### What we learned

During Assignment 6, we learned that a stable and focused MVP is more important than adding many unfinished features. The core value of the project is the telemetry pipeline: Traffic Processor, backend, Docker setup, and dashboard.

Customer feedback helped us prioritize reliability, clear startup instructions, realistic traffic testing, and future database support.

### What went well

- The system can be started through Docker Compose.
- The dashboard shows live traffic data, protocol information, and top IPs.
- The team adjusted the scope and removed unstable features from the final delivery.
- Customer feedback was used to define the next priorities.

### What could be improved

- Deployment and security boundaries should have been defined earlier.
- Realistic traffic testing should have started earlier.
- Documentation should be updated continuously during development.
- The traffic bottleneck still needs deeper investigation.

### Next steps

- Add database storage for historical traffic data.
- Keep dashboard data after refresh.
- Add selectable time windows for graphs.
- Improve configuration for different traffic sources.
